# 交付目录规范

这份规范用于把一次“一稿通”输出整理成清晰的本地交付包。公开版不强制你使用某个固定磁盘路径，下面只给相对目录结构。

## 推荐目录

```text
deliverables/
  YYYY-MM-DD-题材名/
    01-选题侦察与自动选择.md
    02-小红书9页漫画分镜.md
    03-图片worker提示词.md
    03A-每页图片提示词.md
    03B-负面提示词.md
    03C-后期叠字清单.md
    04-小红书发布文案.md
    04-小红书发布文案.html
    05-知乎寓言成稿.md
    05-知乎寓言成稿-图文插入版.html
    06-知乎图文插入建议.md
    06A-知乎发布推荐.md
    07-QA验收清单.md
    08-图片生成状态报告.md
    images/
      无字无水印底图/
      小红书最终上传版/
      qa-contact-sheets/
    zhihu-cover.png
```

- `03-图片worker提示词.md`

## 最终交付必须包含

- 9 张无字底图。
- 9 张最终叠字上传图。
- 每页图片提示词。
- 负面提示词。
- 后期叠字清单。
- 小红书发布文案 Markdown。
- 小红书发布文案 HTML。
- 知乎寓言成稿 Markdown。
- 知乎寓言成稿 HTML 图文插入版。
- 知乎图文插入建议。
- 知乎发布推荐：搜索关键词、3 个推荐话题、1 个投稿问题/投稿方向和选择理由。
- contact sheet。
- QA 检查报告。
- 图片生成状态报告，明确是否实际调用 Image2 / image generation，是否完成无字底图和最终叠字图。

只生成图片或只生成 Markdown 不能判定为完整通过。QA 必须检查 `04-小红书发布文案.html`、`05-知乎寓言成稿-图文插入版.html` 和 `06A-知乎发布推荐.md` 是否真实存在。若小红书最终上传图已经生成，知乎 HTML 图文插入版必须在 `作者的话` 之前，用 `<figure><img></figure>` 或等价结构插入 9 张漫画图；不能只给插图建议。

知乎发布推荐必须包含：搜索关键词、正好 3 个推荐话题、正好 1 个投稿问题/投稿方向、每项选择理由。若无法实时查看知乎发布页或候选热度，应标明“待发布时复核”，不得编造热度数据。

如果当前环境没有图片生成能力，交付包可以只包含图片 worker 提示词、负面提示词、叠字清单和 QA 标准，但必须在 `08-图片生成状态报告.md` 写明“本轮未实际生成图片”。不得伪造图片路径。

## 不建议放进交付包的内容

- 失败图片版本。
- 大量中间图。
- 平台登录状态。
- token、cookie、验证码、二维码。
- 浏览器缓存。
- 与本次题材无关的临时文件。

## 公开分享时的额外清理

如果你要把交付包发给别人，请先检查：

1. 是否包含本机绝对路径。
2. 是否包含账号信息。
3. 是否包含未公开素材。
4. 是否包含内部流程词。
5. 是否包含平台凭据或登录状态。
