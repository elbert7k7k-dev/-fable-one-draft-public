# 5 分钟快速测试

这份测试用于判断你是否能只靠公开仓库跑通“寓言故事一稿通”流程。

## 适合谁

- 第一次打开这个仓库的人。
- 想快速试用提示词的人。
- 想判断这个 Skill / Prompt 是否适合自己内容流程的人。

## 测试前准备

你只需要一个可以联网和读写中文的 AI 对话工具。若工具不能直接读取 GitHub 文件，就手动复制 `prompts/寓言故事一稿通公开版提示词.md` 的全文。

## 最短测试提示词

复制下面这段到新对话：

```text
请只使用这个公开仓库里的工作流进行测试：
https://github.com/elbert7k7k-dev/-fable-one-draft-public

请读取 README 和 prompts/寓言故事一稿通公开版提示词.md。
主题为空，请自动选题，不要让我确认。
如果当前 Codex 环境具备 Image2 / image generation / 图片生成工具，请不要只输出图片提示词，必须实际调用图片工具逐页生成 9 张无字底图，并继续完成 9 张最终叠字上传图。
如果当前环境没有图片生成能力，才输出完整图片 worker 提示词、负面提示词、叠字清单和 QA 标准，并明确说明“本轮未实际生成图片”。
请完成一次小红书 9 页漫画分镜、图片生成或图片 worker 降级材料、小红书发布文案 Markdown 与 HTML、知乎寓言成稿 Markdown 与图文插入版 HTML、知乎发布推荐包和 QA 清单。
知乎发布推荐包必须包含搜索关键词、3 个推荐话题、1 个投稿问题/投稿方向和选择理由。
QA 必须检查小红书发布文案 HTML、知乎图文插入版 HTML、知乎发布推荐包是否真实存在。只生成图片或只生成 Markdown 不能判定为完整通过。
不要发布平台，不要读取账号凭据，不要引用任何本地私有路径。
```

## 预期结果

一次合格测试至少应输出：

1. 已读取哪些公开仓库文件。
2. 10 个候选短词条。
3. 自动选择的 Top 1 题材。
4. 合规性检查。
5. 核心机制。
6. 3 个故事方向和自动选择理由。
7. 小红书 9 页漫画分镜。
8. 图片 worker 提示词和负面提示词。
9. 小红书发布文案 Markdown 与 HTML。
10. 知乎寓言成稿 Markdown 与图文插入版 HTML。
11. 知乎图文插入建议。
12. 知乎发布推荐包：搜索关键词、3 个推荐话题、1 个投稿问题/投稿方向和选择理由。
13. QA 清单。

如果测试环境具备 Image2 / image generation 能力，合格测试还应继续完成：

13. 实际生成 9 张无字底图。
14. 在无字底图基础上完成 9 张最终叠字上传图。
15. 生成 contact sheet 或等价总览检查图。
16. 在 QA 中明确区分“无字底图已生成”和“最终叠字上传图已生成”。
17. 明确说明没有用 SVG、ASCII、本地脚本绘制、PPT 图、知识卡片图或占位图冒充最终图。
18. 真实存在小红书发布文案 HTML。
19. 真实存在知乎寓言成稿 HTML 图文插入版。
20. QA 中明确检查上述两个 HTML 文件存在。
21. 真实存在知乎发布推荐包。
22. QA 中明确检查知乎发布推荐包包含搜索关键词、3 个话题和 1 个投稿方向。

如果测试环境没有图片生成能力，才允许只输出图片 worker 提示词，并且必须明确说明“当前环境未实际生成图片”。

## 判断是否跑通

如果 AI 停在“请你选择题材”，说明自动化规则没有生效。

如果 AI 引用你的本机路径、私有飞书页面或历史对话，说明它没有按外部用户视角执行。

如果 AI 用简笔人、PPT 图、知识卡片或脚本占位图冒充最终图片，说明图片规则没有生效。

如果 AI 明确说明无法读取 GitHub，但仍能在你手动复制提示词后跑完，说明问题在仓库访问能力，不在提示词本身。

如果 AI 只生成了无字底图，没有继续生成带中文旁白/对话的最终叠字图，说明 v1.2 图片交付规则没有执行完整。

如果 AI 只生成图片、只生成 Markdown，或没有创建小红书发布文案 HTML、知乎图文插入版 HTML、知乎发布推荐包，说明完整交付规则没有执行完整。此时只能判定为“图片链路通过，完整交付未通过”。
