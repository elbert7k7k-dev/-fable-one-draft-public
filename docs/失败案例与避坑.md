# 失败案例与避坑

这份文档记录公开版最容易出现的失败模式，以及如何修正。

## 失败 1：停在选题确认

表现：

AI 输出 10 个候选词条后，让用户选择一个。

原因：

旧式人工协作流程覆盖了自动化规则。

修正：

在提示词中强调：

```text
主题为空时自动选题，候选题材只作为记录，不作为暂停点。
请自行选择 Top 1 并继续执行。
```

## 失败 2：漫画变成知识卡片

表现：

图片提示词像 PPT、海报、信息图或概念解释卡。

原因：

没有把“故事场景、人物动作、生活物件”写具体。

修正：

每页必须写清：

- 谁在场。
- 在什么现实生活场景。
- 发生了什么动作。
- 关键物件是什么。
- 情绪如何变化。

## 失败 3：用脚本占位图冒充最终图

表现：

图片是几何图形、简笔人、符号人或抽象示意图。

原因：

把“自动生成图片”误解为“本地脚本快速画图”。

修正：

明确要求：

```text
本地脚本只允许叠字、水印、裁切、尺寸检查和 contact sheet。
不得用脚本画人物、主场景或关键物件。
```

## 失败 4：字幕出现白底框或黑底条

表现：

画面上出现大块白色字幕框、黑色字幕条或半透明补丁。

原因：

后期叠字为了可读性过度加底托。

修正：

默认使用无字底描边字，只给文字加描边和轻微阴影。文字放在天空、墙面、桌面、地面或边角留白处。

## 失败 5：知乎文章变成解释文

表现：

知乎正文一开始就讲概念、定义和方法论，中途很多解释型小标题。

原因：

没有遵守“先讲完整寓言故事，再写作者的话”。

修正：

知乎结构固定为：

1. 故事体标题。
2. 完整寓言故事。
3. 一个“作者的话”。

## 失败 6：假装已经有图

表现：

没有实际图片，却在 HTML 或文案中写入虚假的图片路径。

原因：

把“插图建议”误写成“已生成图片”。

修正：

如果图片尚未生成，只输出插图清单和待回填说明，不要写假路径。

## 失败 7：引用私有路径

表现：

输出中出现本机绝对路径、私有飞书链接、历史项目记忆或账号环境。

原因：

没有按外部用户视角执行。

修正：

公开版测试时只允许使用公开 GitHub 仓库，不允许引用本机路径和私有资料。
