# 寓言故事一稿通公开版

这是一个面向 Codex / AI 写作工作流的公开版 Skill + Prompt 包，用来把中文短词条、社会现象词、平台黑话、行业术语或流行短语，转译为：

1. 小红书 9 页叙事漫画分镜。
2. 小红书发布文案。
3. 知乎寓言故事成稿。
4. 图片生成和后期叠字的质量规则。

核心策略是：漫画先行。先用小红书漫画把故事讲清楚，再用知乎文章把机制讲透。

## 快速开始

第一次试用建议按这个顺序看：

1. `docs/5分钟快速测试.md`
2. `prompts/寓言故事一稿通公开版提示词.md`
3. `examples/完整脱敏示例输出.md`

如果你只想马上测试，打开 `docs/5分钟快速测试.md`，复制里面的最短测试提示词到一个新对话即可。

## 包含内容

- `prompts/`：可直接复制到新对话使用的完整提示词。
- `skills/`：Codex Skill 形式的公开工作流说明。
- `templates/`：小红书漫画、知乎成稿、发布包模板。
- `docs/`：安装、使用、工作流、图片落地和避坑说明。
- `examples/`：脱敏示例，帮助理解输出形态。

## v1.3 Image2 强触发规则

在 Codex 或其他具备 Image2 / image generation 能力的环境中，本公开版默认必须直接调用图片工具完成图片阶段，而不是只输出图片提示词或图片 worker 任务单：

1. 逐页调用图片工具生成 9 张无字、无水印、无字幕框的漫画底图。
2. 每张为 1024x1536 竖版，2 列 x 3 行六格漫画页。
3. 底图必须是真人现实生活场景漫画，不是知识卡片、PPT、海报、简笔人或符号人。
4. 底图生成后必须继续做中文后期叠字，输出 9 张最终叠字上传图。
5. 如果当前环境没有图片生成工具，才降级输出图片 worker 提示词、负面提示词、叠字清单和 QA 标准。

完成条件不是“已经有 9 张无字底图”，而是“9 张无字底图 + 9 张最终叠字上传图 + contact sheet + QA 报告”全部完成。只停在无字底图即判定为未完成。

最终交付必须明确区分：

- `无字底图`：纯画面页，不含中文、不含水印、不含字幕框。
- `最终叠字上传图`：在无字底图基础上加入中文旁白/对话、必要水印和检查后的发布图。

外部用户第一次测试时，若使用 Codex 且当前会话有图片生成能力，直接复制 `docs/5分钟快速测试.md` 的测试提示词即可触发图片阶段。若 AI 回复“我只能给提示词”但当前环境实际有图片生成工具，应把它视为失败，并要求它按本节重新调用图片工具。

## 不包含内容

本公开版不包含私人项目记忆、飞书页面、真实交付图片、账号凭据、本地浏览器状态、商业规划、未公开案例或任何 token/cookie/验证码/二维码。

## 推荐用法

普通用户可以直接复制 `prompts/寓言故事一稿通公开版提示词.md` 到新对话，让 AI 自动完成选题、分镜、文案和知乎成稿。

Codex 用户可以把 `skills/fable-one-draft-public` 复制到本机的 `.codex/skills/` 目录，然后在相关任务中调用这个 Skill。

## 安全边界

本工作流只生成内容、模板和本地交付建议，不自动发布小红书、知乎或任何平台，不读取账号凭据，不点击发布按钮。
